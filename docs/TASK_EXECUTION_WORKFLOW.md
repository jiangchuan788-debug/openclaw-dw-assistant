## 任务一：异常调度自动检测

### 基本信息

| 项目 | 值 |
|------|-----|
| **任务ID** | `1d37efe1-d348-476c-bf41-1c6b6a78d549` |
| **执行频率** | 每半小时（00分、30分） |
| **执行脚本** | `core/auto_stop_abnormal_schedule.py` |
| **超时时间** | 300秒（5分钟） |
| **TV通知** | 仅发现异常时发送 |
| **钉钉通知** | 已禁用 |

### 执行步骤清单

```
┌─────────────────────────────────────────────────────────────┐
│                    异常调度检测执行流程                       │
└─────────────────────────────────────────────────────────────┘

【步骤1】加载调度配置 (必须)
├─ 读取CSV文件: dolphinscheduler/schedules_export.csv
├─ 加载所有调度配置（28个）
├─ 统计ONLINE/OFFLINE数量
└─ 输出: 配置加载状态

【步骤2】获取运行中实例 (必须)
├─ 调用DS API: /projects/{code}/process-instances
├─ 过滤: stateType=RUNNING_EXECUTION
├─ 多页查询（确保获取全部）
└─ 输出: 运行中实例数量

【步骤3】异常检测 (必须)
├─ 遍历每个运行中实例
│   ├─ 获取实例详情（查看commandType）
│   ├─ 跳过非SCHEDULER类型的实例
│   └─ 检查是否为异常:
│       ├─ 情况A: 无定时配置但被调度启动
│       └─ 情况B: 调度已下线但仍被启动
└─ 输出: 异常实例列表

【步骤4】自动停止异常实例 (条件执行)
├─ 如果有异常实例:
│   ├─ 遍历每个异常实例
│   │   └─ 调用DS API: /executors/execute (STOP)
│   └─ 记录停止结果（成功/失败）
└─ 如果没有异常实例: 跳过此步骤

【步骤5】TV通知 (条件执行)
├─ 如果有异常实例（无论停止成功与否）:
│   └─ 发送TV通知
└─ 如果没有异常实例: 静默，不发送任何通知

【步骤6】生成钉钉报告 (已禁用)
├─ 当前配置: mode=none（不发送）
└─ 仅控制台输出

【步骤7】保存检测记录 (必须)
├─ 保存JSON记录到本地文件
├─ 记录内容: 检测时间、统计信息、异常列表
└─ 输出: 记录文件路径

【步骤8】执行完成 (必须)
└─ 返回执行结果统计
```

### 关键检查点

| 检查点 | 预期结果 | 失败处理 |
|--------|---------|---------|
| CSV文件存在 | 是 | 报错退出 |

| API调用成功 | 是 | 报错退出 |
| 异常检测完成 | 是 | 继续执行 |
| 记录保存成功 | 是 | 控制台警告 |

## 任务二：智能告警修复

### 基本信息

| 项目 | 值 |
|------|-----|
| **任务ID** | `eb6275ba-12de-405a-982f-9065817816c9` |
| **执行频率** | 每日早上 06:40 |
| **执行脚本** | `core/repair_strict_7step.py` |
| **超时时间** | 1800秒（30分钟） |
| **TV通知** | 始终发送（简化版统计） |
| **钉钉通知** | 已禁用 |

### 执行步骤清单

```
┌─────────────────────────────────────────────────────────────┐
│                    智能告警修复执行流程                       │
└─────────────────────────────────────────────────────────────┘

【步骤1】扫描告警 (必须)
├─ 查询数据库: wattrel_quality_result表
├─ 过滤条件: status=0 (未恢复)
├─ 单次执行内去重（同表只保留一个告警）
├─ 输出告警列表
└─ 输出: 扫描结果统计

【步骤2】查找工作流位置 (必须)
├─ 对每个告警表:
│   └─ 调用search_table.py查找工作流和任务
├─ 记录表位置映射
└─ 输出: 映射关系

【步骤3】执行修复 (必须)
├─ 对每个待修复表:
│   ├─ 限制检查:
│   │   ├─ dt范围检查 (±10天)
│   │   ├─ 工作流状态检查 (是否空闲)
│   │   └─ 单次执行内去重 (同表只修复一次)
│   ├─ 调用DS API启动任务 (TASK_ONLY模式)
│   └─ 记录修复结果
└─ 输出: 修复结果列表

【步骤4】记录+复验 (必须)
├─ 4.1 记录重跑次数 (仅统计，不限制)
├─ 4.2 执行复验工作流 (智能选择)
│   ├─ 根据告警智能选择复验:
│   │   ├─ dwb_开头 → 1级表复验 + 每日全级别
│   │   └─ 其他表 → 3级表复验 + 每日全级别
│   └─ 默认必跑: 每日全级别
├─ 4.3 等待复验完成
└─ 4.4 再次检查告警

【步骤5】发送钉钉报告 (已禁用)
├─ 当前配置: mode=none（不发送）
└─ 仅控制台输出

【步骤6】保存操作记录 (必须)
├─ 保存3类文件:
│   ├─ .json 结构化数据
│   ├─ .log 执行日志
│   └─ .sh 重跑命令脚本
└─ 输出: 文件路径

【步骤7】发送TV报告 (必须)
├─ 发送简化版修复统计
└─ 无论成功与否都发送

【步骤8】执行完成 (必须)
└─ 返回执行结果统计
```

### 关键检查点

| 检查点 | 预期结果 | 失败处理 |
|--------|---------|---------|
| 数据库连接 | 成功 | 报错退出 |

| 复验执行 | 部分可能失败 | 继续执行 |
| 记录保存 | 成功 | 控制台警告 |
| TV发送 | 成功 | 控制台警告 |

## 检查清单

### 每次执行前检查

- [ ] 环境变量已设置（DS_TOKEN, DB_PASSWORD）
- [ ] CSV文件存在且可读
- [ ] 数据库可连接
- [ ] DS API可访问

### 每次执行后检查

- [ ] 步骤无遗漏（查看控制台输出）
- [ ] 记录文件已生成
- [ ] TV通知已发送（如有需要）
- [ ] 无未处理异常

## 常见问题

### Q1: 步骤执行顺序是否固定？
**A**: 是的，必须按1→2→3→4→5→6→7→8顺序执行，不可跳过。

### Q2: 某步骤失败后是否继续？
**A**: 
- 关键步骤（1,2）失败 → 退出
- 非关键步骤（3,4）失败 → 继续，但标记失败
- 记录步骤（6,7）失败 → 继续，控制台警告

### Q3: 如何确认执行完整？
**A**: 查看控制台输出是否包含"✅ 8步流程全部完成"或"✅ 异常调度检测完成"。
