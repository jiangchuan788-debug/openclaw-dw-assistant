### 补充1: dt提取和验证规则

**用户示例**:
```
【执行语句】
```

**提取逻辑**:

### 补充2: 并行控制机制

**约束条件**:
1. **同工作流串行**: DWD工作流一次只能跑一个表
2. **全局并行限制**: 最多同时跑2个任务

**实现方案**:
```python
# 信号量控制全局并行
semaphore = threading.Semaphore(2)

# 工作流锁控制同工作流串行
workflow_locks = {
    '158514956979200': False,  # DWD工作流
    '158514957297664': False,  # DWB工作流
    ...
}

# 执行流程
with semaphore:  # 获取全局许可
    while workflow_locks[wf_code]:  # 等待同工作流任务完成
        sleep(30)
    workflow_locks[wf_code] = True  # 加锁
    try:
        execute_task()  # 执行任务
    finally:
        workflow_locks[wf_code] = False  # 解锁
```

## 已实现的防护机制

| 防护点 | 实现方式 | 说明 |
|-------|---------|------|
| dt范围检查 | `0 <= diff_days <= 10` | 防止重跑过旧或未来数据 |
| 工作流状态检查 | `check_running.py` | 避免冲突 |
| 同工作流串行 | `workflow_locks` | 防止同一工作流并发 |
| 全局并行限制 | `Semaphore(2)` | 控制资源使用 |
| 执行前确认 | 打印执行计划 | 人工检查dt值 |
| 超时保护 | `timeout=35s` | 防止API卡死 |
| 失败重试 | 异常捕获 | 记录失败原因 |
